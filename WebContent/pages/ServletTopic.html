The Servlet Tutorial

WebServer

To host a website with only static pages webserver is needed.The webserver parses the request from client/browser and it navigates to the requested page. The webserver is configured to parse specific type of request. Most used request or protocol type is HTTP.

Webserver is also called HTTP server since it uses HTTP to communicate with client(web browser). The HTTP request consist of following :
  • Method-URI-Protocol
  • Request headers
  • Entity Body
Webserver uses two important classes:
  • java.net.Socket
  • java.net.ServerSocket
The above classes keep checking on a port if a request has come on that port using socket of networking


Application Server

To apply business logic for a request, the call need to be transferred to application server from webserver. Application server is a software which installs J2E specifications. an application server refers to a framework (servlet container, EJB container, JSP engine, MQ container, etc.) for handling Web applications. The app server may also interact with database as per requirement. Application container also contains enterprise tools.

Servlet Container

A servlet is nothing but Java code that runs in a container. It generates HTML & other contents that get displayed in the web page that we see. It is purely coded in Java, so the benefits and restrictions of all regular Java classes apply here too.

The servlet container is an extension of a Web server in the same way CGI, ASP, and PHP are. A servlet functions just like them, but the language in which it is written is Java. That's the main difference. The servlet doesn't talk to the client directly. The Web server functions as the intermediary that does it for the servlet. In a chain of processes, the client sends a request to the Web server, which hands it to the container, which hands it to the servlet. The Servlet processes the request and generates a response. The response starts from the servlet and goes to the container and then to the Web server and back to the client. Of course there are several other steps that happen too but this is just the introduction so this much would suffice I believe.

How Servelet Works

It is important to learn how servlet works for understanding the servlet well. Here, we are going to get the internal detail about the first servlet program.
The server checks if the servlet is requested for the first time.
If yes, web container does the following tasks:
  • loads the servlet class.
  • instantiates the servlet class.
  • calls the init method passing the ServletConfig object
else calls the service method passing request and response objects The web container calls the destroy method when it needs to remove the servlet such as at time of stopping server or undeploying the project.

The protected service method checks the type of request, if request type is get, it calls doGet method, if request type is post, it calls doPost method, so on
